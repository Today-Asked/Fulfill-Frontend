請幫我設計一套 mobile-first 的創作者委託平台 UI，產品名稱為 Fulfill。
這是一個讓「委託方」與「創作者」進行作品委託、溝通、下單、追蹤訂單與管理作品集的平台。

整體風格請做成：

深色系、精緻、年輕、偏設計社群感

視覺方向介於 Behance / Instagram / commission marketplace

使用 glassmorphism + soft gradient accent

主色調為：

背景：#0A0A0F

卡片底：半透明深灰 / 黑

品牌色：violet / fuchsia / cyan

視覺語言：

大圓角卡片

柔和陰影

半透明描邊

清楚的資訊層級

現代感 icon

字體風格：

乾淨、現代、偏 Inter / Noto Sans TC 類型

版面尺寸：

手機 frame 為 390 x 844

請使用 Auto Layout

請建立可重複元件與一致的 spacing / typography / color styles

請優先產出 可編輯的 mobile UI screens

請設計以下頁面：

1. Onboarding / Register

手機註冊與初始設定頁，內容包含：

標題：創作者委託平台

使用者身份切換：

上傳頭像 / 作品縮圖區塊

請做成有品牌感的 onboarding screen，簡潔但有未來感。

首頁是探索與推薦頁，包含：

Header：頁面標題、通知 icon、menu icon

Search bar：搜尋創作者、作品風格、關鍵字

Filter button

推薦 tag：

卡片大小有大中小變化

每張卡片包含 tag、標題、簡短描述

主題像是：

霧面藍銀角色設計

Y2K 專輯封面

極簡刺青線稿

手作雕塑質感

狀態 badge（Open for Commission / Limited Slot / Fast Reply）

CTA：

看作品集

一鍵下單
請讓首頁具有「靈感探索 + 下單轉換」雙重目的。

3. Create Post

建立委託需求頁，風格像 Instagram 發文結合委託表單。
包含：

頁面標題：新增 / Post

圖片拖曳上傳區

標題輸入欄

多行需求描述欄位

預算欄位

希望交付日期欄位

主按鈕：「發佈需求」

下方或側邊有 3 步驟說明卡片：

選照片 / 參考圖

打內文與委託條件

Post 後進入聊天與訂單
請把這頁做得很清楚，讓使用者知道這不是一般貼文，而是可進入委託流程的需求單。

due date

對話列表

對話主畫面

聊天對象資訊列

聊天語氣是委託討論，不是一般社群聊天。
請讓畫面看起來專業、乾淨、容易閱讀。
